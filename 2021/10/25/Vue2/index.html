一些关于Vue2的笔记

基于标准 HTML、CSS 和 JavaScript 构建，并提供了一套声明式的、组件化的编程模型

核心功能

1、声明式渲染 2、数据驱动视图更新

场景需求

1、单页应用 (SPA) 2、全栈 / 服务端渲染 (SSR) 3、Jamstack / 静态站点生成 (SSG)

api风格

选项式 API (Options API)

构建工具

Webpack

为什么使用webpack？

  • 模块化，让我们可以把复杂的程序细化为小的文件;
  • 类似于TypeScript这种在JavaScript基础上拓展的开发语言：使我们能够实现目前版本的JavaScript不能直+ 接使用的特性，并且之后还能转换为JavaScript文件使浏览器可以识别；
  • Scss，less等CSS预处理器

功能点

生产环境配置 webpack.config.js

  • 配置devtool，生成Source Maps（使调试更容易）
  • 配置devserver，构建本地服务器
  • 配置Loaders，转换scss为css，（ES6，ES7)转换为现代浏览器兼容的JS文件，JSX文件转换为JS文件
  • 配置Babel，编译JavaScript
  • 配置CSS module
  • 插件（Plugins） new xxx

开发环境配置 webpack.production.config.js

  • 优化插件：为组件分配ID，通过这个插件webpack可以分析和优先考虑使用最多的模块，并为它们分配最小的ID；压缩JS代码；分离CSS和JS文件

性能优化

缓存

缓存的本质是首轮计算后将结果保存下来，下次直接复用计算结果而跳过计算过程

  • webpack5 使用cache 功能,使用持久化缓存 cache.type = 'filesystem' 即可开启
  • Webpack 4 及之前使用一些 loader自带的缓存功能提升构建性能，例如 babel-loader、eslint-loader、cache-loader
  • require.context() 常常用来在组件内批量导入多个组件

多线程打包

并行的本质则是在同一时间内并发执行多个运算，提升单位时间计算效率
受限于 Node.js 的单线程架构，原生 Webpack 对所有资源文件做的所有解析、转译、合并操作本质上都是在同一个线程内串行执行，CPU 利用率极低，因此，理所当然地社区出现了一些基于多进程方式运行 Webpack

方案：使用 Thread-loader
使用方法：首先，需要安装 Thread-loader 依赖：
yarn add -D thread-loader
其次，需要将 Thread-loader 配置到 loader 数组首位，确保最先运行
原理：创建工作子进程，并将Loader 列表、文件路径、上下文等参数传递到子进程，子进程中调用 loader-runner，转译文件内容,转译完毕后，将结果传回主进程

Vite

为什么使用Vite？

vite服务器启动速度比webpack快，由于vite启动的时候不需要打包，也就无需分析模块依赖、编译，所以启动速度非常快。当浏览器请求需要的模块时，再对模块进行编译，这种按需动态编译的模式，极大缩短了编泽时间，
当项目越大，文件越多时，vite的开发时优势越明显。vite热更新比webpack快，vite在HRM方面，当某个模块内容改变时，让浏览器去重新请求该模块即可，而不是像webpack重新将该模块的所有依赖重新编译
